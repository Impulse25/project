# Изменения по задачам РУПЛ и ведомостям

1. Импорт РУПЛ теперь забирает код специальности, наименование специальности и квалификацию из листа `Паспорт`.
   В проекте используется существующая колонка `specialty_code`, а не ошибочно написанная `speciality_code`.

2. В `curriculum_import.php` исправлен расчёт срока обучения: считаются только реальные семестры 1–8 и игнорируются строки `Итого`.
   Для приложенного шаблона срок обучения должен определяться как 4 года, а не 6.

3. Добавлена миграция `migrations/002_curriculum_passport_graph_summary.sql`:
   - `edu_curriculum_passport_fields`;
   - `edu_curriculum_process_schedule`;
   - `edu_curriculum_summary`.

4. В просмотре РУПЛ (`curriculum_view.php`) добавлены вкладки:
   - Паспорт;
   - Учебный план;
   - График учебного процесса;
   - Сводные данные;
   - Компетенции;
   - Группы.

5. Страница `Сформировать ведомость` (`vedomost_generate.php`) получила режимы:
   - выбранная дисциплина — старый DOCX-сценарий;
   - все дисциплины семестра — XLSX с листом группы и листом студентов на стипендию;
   - все 8 семестров — XLSX с такими листами по каждому семестру.

6. При формировании ведомостей специальность и квалификация берутся из РУПЛ, привязанного к группе.

7. Страницы `Учебные планы (РУПл)` и `Сформировать ведомость` приведены к одному стилю заголовков/карточек/действий.

## После установки

Откройте под администратором:

`edu/run_migration.php`

После миграции повторно импортируйте РУПЛ. Старые импортированные планы не заполнят новые вкладки автоматически.

3. В БД добавлены/обновлены структуры:
   - `edu_curriculum_process_schedule.span_weeks`;
   - `edu_curriculum_process_legend`.
4. На страницах `Учебные планы (РУПл)` и `Сформировать ведомость` добавлена/заменена кнопка возврата на главную страницу модуля.

После установки снова откройте `edu/run_migration.php`, затем повторно импортируйте РУПЛ.
